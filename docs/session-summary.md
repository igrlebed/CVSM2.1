## 1. Контекст и цель

Цифровая модель ВСМ (CVSM2.1) — прототип системы прогнозирования эффектов от реализации проектов развития сети высокоскоростных железнодорожных магистралей ОАО «РЖД».

**Цель сессии:** Довести текущий прототип до соответствия ТЗ (27768025.10985.501.ТЗ от 18.03.2026) на уровне информационной архитектуры, ролевой модели, пользовательских экранов и сценариев.

**Подход:** Эволюционная переработка существующего кода без сноса рабочей карты (SVG-слой map не трогается).

**Было:** 4 роли — `operator`, `analyst`, `approver`, `admin`

- Ссылки: Обзор → Карта → Проекты → Конструктор

## 6. Как работать с текущей веткой

```bash
# Переключиться на рабочую версию (стабильную)
git checkout main

# Переключиться на версию с правками
git checkout feat/tz-alignment-v2

# Синхронизировать с удалённым репозиторием
git pull origin feat/tz-alignment-v2

# Вернуться к стабильной версии (тег)
git checkout v1.0-stable
```

**Для деплоя на Vercel:**
- Ветка `main` → production домен
- Ветка `feat/tz-alignment-v2` → preview URL (автоматически при push)

**Для демонстрации заказчику:**
1. Войти как `ivanov` (Эксперт) — показать основной пользовательский опыт
2. Войти как `admin` (Администратор) — показать администрирование
3. Показать «Петрова А.С.» — сценарий первого входа со сменой пароля
4. Показать «Сидоров К.В.» — сценарий блокировки (роль истекла)
