/analytics                  — Аналитика (обзор)
/analytics/dashboards        — Дашборды

## 4. Пользовательские потоки

### 4.1. Авторизация
```
/ → проверка auth
  ├── authenticated → /
  └── не authenticated → /auth/login
       ├── успех → проверка firstLogin
       │    ├── true → /auth/first-login → /
       │    └── false → /
       ├── 5 неудачных попыток → /auth/locked (30 мин)
       └── забыли пароль → /auth/forgot-password
```

### 4.2. Создание проекта
```
/projects → [Создать] → /projects/new
  → заполнение формы → валидация
    ├── успех → /projects/[id] (toast: «Проект создан, код: ERR-SUCCESS-001»)
    └── ошибка → форма с подсветкой полей (код ошибки + расшифровка)
```

### 4.3. Экспорт (асинхронный)
```
/projects → [Экспорт] → модальный выбор формата
  → задача в очередь → toast: «Экспорт запущен»
  → фон: завершение → toast: «Экспорт готов» + ссылка на скачивание
```

### 4.4. Ранжирование
```
/analytics/ranking
  → выбор показателей → установка весов → правила предпочтения
  → пороги отсечки → [Рассчитать]
  → ранжированный список → [Сохранить сценарий]
```

### 4.5. Администрирование
```
/admin/users → [Создать] → /admin/users/new
  → ФИО, логин, подразделение, должность, email, роли, дата окончания
  → генерация временного пароля → уведомление
  → пользователь входит → /auth/first-login → смена пароля

/admin/journal → фильтры по дате/типу/пользователю → просмотр (read-only) → [Экспорт xlsx]

/admin/audit → выбор пользователя → timeline событий (вход/выход, транзакции, изменения ролей)
```
